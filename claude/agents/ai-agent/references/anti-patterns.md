# Skills/子代理反模式

创建或修改时应避免的常见错误。

## 1. Quick Reference 变成文档堆砌

**症状**: 包含大段粘贴的文档
**修复**: 移至 `references/`，Quick Reference 只保留可复制模式

## 2. 触发条件模糊

**症状**: description 写"帮助处理X"
**修复**: 改为"做什么+何时触发"，列出可判定任务

## 3. 缺少边界定义

**症状**: 不说明不做什么
**修复**: 添加 `Not For / Boundaries`

## 4. 示例不可复现

**症状**: 伪代码、缺输入/输出
**修复**: 每个示例含输入→步骤→预期输出

## 5. 单文件综合症

**症状**: 所有内容在一个文件
**修复**: 拆分到 `references/`，入口保持执行导向

## 6. 编造事实

**症状**: 声称API/命令不引用来源
**修复**: 添加验证路径，标记假设

## 7. 不安全默认操作

**症状**: 破坏性命令作为默认
**修复**: 警告+确认步骤，优先只读诊断

## 8. 术语不一致

**症状**: 同概念多名称
**修复**: 添加术语表，一概念一名称

## 9. 流程不完整

**症状**: 不说明完成后做什么
**修复**: 添加完成流程（报告/commit/record）

## 10. 硬性规则缺失

**症状**: 无必须遵守的规则
**修复**: 添加硬性规则（禁止项/格式/语言）
