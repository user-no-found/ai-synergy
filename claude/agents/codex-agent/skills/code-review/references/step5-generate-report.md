# 步骤5：生成审核报告

## 报告位置

`Record/review/{plan_version}-review.md`

## 报告内容

详见 `review-report-template.md`

## 追加项目记录

审核完成后，追加记录到 `Record/record.md`：

```markdown
## YYYY-MM-DD HH:MM 代码审核
状态：{通过/不通过}
问题数：{问题数量}
高风险：{高风险问题数}
```

## 审核结论

### 审核通过

如果没有高风险问题：
- 输出审核报告
- 更新所有提案状态为 `accepted`
- 告知用户审核通过
- 等待用户确认（接受/部分接受/强制通过）

### 审核不通过

如果存在高风险问题：
- 输出审核报告（含问题清单）
- 保持提案状态为 `review`
- 告知用户需要修复
- 为每个需要修复的问题创建修复提案（见步骤6）
