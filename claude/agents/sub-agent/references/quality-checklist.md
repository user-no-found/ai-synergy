# 子代理质量门禁（Quality Gate）

创建或修改子代理时使用此清单评估质量。

## 评分标准

每项评分：
- 2 = 完全满足
- 1 = 部分满足/需改进
- 0 = 缺失

**发布阈值**：总分 >= 24（满分32），关键项不得低于2分

## A. 激活可靠性（关键）

| 项目 | 分值 | 检查点 |
|-----|------|--------|
| 1 | 2 | frontmatter `name` 匹配 `^[a-z][a-z0-9-]*$` 且与文件名一致 |
| 2 | 2 | frontmatter `description` 明确说明"做什么+何时触发" |
| 3 | 2 | `## When to Use This Skill` 列出具体任务/输入/目标 |
| 4 | 2 | `## Not For / Boundaries` 存在且有效防止误触发 |

## B. 可用性（关键）

| 项目 | 分值 | 检查点 |
|-----|------|--------|
| 5 | 2 | `## Quick Reference` 简洁可用（非文档堆砌） |
| 6 | 2 | Quick Reference 中的模式可直接复制使用 |
| 7 | 2 | `## Examples` 包含 >= 3 个可复现示例 |
| 8 | 2 | 示例包含验收标准/预期输出 |

## C. 证据与正确性

| 项目 | 分值 | 检查点 |
|-----|------|--------|
| 9 | 2 | `## Maintenance` 标注来源和最后更新日期 |
| 10 | 2 | 不确定的外部细节包含验证路径 |
| 11 | 2 | 术语一致（一个概念一个名称） |
| 12 | 2 | Quick Reference 和 Examples 之间无矛盾 |

## D. 结构与可维护性

| 项目 | 分值 | 检查点 |
|-----|------|--------|
| 13 | 2 | 长文档放在 `references/` 并有 `index.md` 导航 |
| 14 | 2 | 参考文件按主题拆分（非单一大文件） |
| 15 | 2 | 子代理读起来像操作手册（任务→步骤→验收） |
| 16 | 2 | scripts/assets 精简且职责明确（可选） |

## 常见不通过原因

1. 激活条件模糊（"帮助处理X"）无边界
2. Quick Reference 是粘贴的文档而非模式
3. 示例不可复现（无输入、无步骤、无预期输出）
4. 无来源和更新日期（无法信任或维护）

## 快速检查清单

```
[ ] name 格式正确且与文件名匹配
[ ] description 包含"做什么+何时触发"
[ ] 有明确的触发条件列表
[ ] 有明确的边界定义
[ ] Quick Reference 可直接使用
[ ] 至少3个可复现示例
[ ] 有来源和更新日期
[ ] 术语一致无矛盾
```
